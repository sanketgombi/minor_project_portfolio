Design Constraints

Requirements & Specifications

Functional Requirements

  • Securely empty plastic & cardboard bins up to 1.5 kg.
  • Passive compliance to prevent deformation of delicate payloads.
  • Repeatable docking alignment within ±2.0 mm tolerance.

Physical Constraints

  • Total end-effector weight < 1.5 kg (preserving robot payload).
  • Maximum grasping span: 110 mm.
  • Corrosion-resistant materials (PLA, TPU).

Electrical Specifications

  • Input actuation voltage: 24V DC.
  • Integration with dual FSR force feedback sensors.
  • Direct power draw from TM5 robot wrist port (< 1.5A).

Control Interface

  • Modbus TCP communication protocol.
  • Handshake interface with ASRS PLC controller.
  • Integration with OMRON TM Flow camera vision landmarks.

Selected Solution

Finalized Concept Details

Optimized Hybrid Soft-Claw

After rigorous feasibility testing and decision matrix analysis, we integrated the Hybrid Soft-Claw design. This end-effector provides the perfect balance between the payload capacity required for ASRS redocking and the gentle touch needed for mixed-item pallet emptying workflows without causing surface damage.

  • ✓ Lightweight 3D printed PLA core ensures TM5 payload remains under 6kg cap.
  • ✓ Integrated force sensors for safe compliance feedback.
  • ✓ Seamless Modbus TCP I/O interface directly with the TM5-700 controller.

Engineering Blueprints

SolidWorks Drawings

Detailed engineering blueprint layouts were drafted in SolidWorks to establish geometrical dimensions, mounting flange adapter configurations conforming to ISO 9409-1-50-4-M6, and pin clearance margins.

Finite Element Method

System Analysis (FEA)

Structural stress and displacement simulations were conducted on the TPU gripper finger elements using finite element analysis. A maximum design load of 15N was applied, confirming a safety factor of 2.4 with maximum stress well below the PLA/TPU yield strength limits.

System Electronics & Control Logic

Control Systems Engineering

Circuit Design

The electrical circuit interfaces the robot's wrist GPIO port (24V, 1.5A supply) with a relay driving the 24V linear actuator. Two Force Sensitive Resistors (FSRs) are connected in voltage-divider circuits back to the controller's analog inputs to provide real-time grasping pressure metrics.

[Cobot Wrist 24V] ---> [Relay Controller] ---> [Linear Actuator]
[FSR Sensors] ---> [Analog Dividers] ---> [Wrist AI 1 & 2]

MATLAB Architecture

Feedback control loop developed in MATLAB Simulink. Consists of a PID position controller driving the actuator spool dynamics, feedback comparison block, sensor noise filters, and saturation limits to prevent actuator overtravel.

Simulation Results

We simulated the transient response of the linear actuator's closed-loop position system in MATLAB Simulink. The simulation confirms smooth trajectory tracking under variable cargo weight loads (up to 1.5 kg). Overshoot was restricted to less than 4% with a settling time under 0.85 seconds, matching our strict cycle time design criteria.

Physical Build & Integration

Hardware & Software Execution

1. Additive Manufacturing

The structural flange adapter and core mechanical links were printed in PLA with a 60% triangular infill, 4 shells, and 0.2mm layer height to ensure structural rigidity. The adaptive compliant finger tips were printed using flexible TPU (95A Shore hardness) with a 20% gyroid infill to enable passive surface conformity.

2. Actuator Integration

A 24V micro linear actuator (100mm stroke length, 150N max push force) was bolted to the core links. A linear sliding guide rail was integrated to convert the actuator's linear travel into pure translational jaw movement, preventing binding under off-axis torque.

3. Sensor Assembly

Dual Force Sensitive Resistors (FSR 402) were adhered to the inner compliant TPU pads. The sensors were wired to a local junction terminal box, providing Analog inputs directly into the TM5 cobot wrist GPIO port, shielded against electromagnetic interference.

TM Flow Scripting & Control Loop

We programmed the robot motion and control sequence using the OMRON TM Flow visual software. The cobot continuously polls the digital inputs representing target cargo presence from ASRS sensors.

Key Script Milestones:
  • Landmark Detection: Dynamic calibration of docking targets using the built-in wrist camera.
  • Compliance Poll: Gripper actuator halts once FSR reading reaches a threshold of 2.2V (representing 10N force).
  • Modbus Handshake: Modbus TCP coil commands coordinate shelf slot status with the central PLC database.
// Pseudo-code of Gripper Force Control Thread
#define FSR_THRESHOLD 450 // ~10N grip force
#define WRIST_GPIO_RELAY 0x04

void grasp_sequence() {
  // Engage linear actuator relay
  set_digital_output(WRIST_GPIO_RELAY, HIGH);

  while(get_analog_input(0) < FSR_THRESHOLD) {
    // Loop & wait for target gripping pressure
    sleep_ms(10);
    if (read_limit_switch()) break;
  }

  // Hold position by shutting off motor power
  set_digital_output(WRIST_GPIO_RELAY, LOW);
}

ASRS Workspace Calibration

We mounted OMRON visual landmark tags on the ASRS docking rack. By utilizing the TM5-700's camera sensor, the system dynamically calculates coordinate transformations to compensate for small shifts in the rack alignment, ensuring precise redocking.
